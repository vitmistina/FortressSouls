# Fortress Souls v0.1 Backlog

Master backlog for v0.1 implementation. Product behavior and boundaries are in
`docs/specs/fortress-souls-v0.1.spec.md`.

Mini-specs refine a backlog item for execution; they do not supersede this
backlog, accepted ADRs, or product contracts. Start implementation from the
linked mini-spec. Preserve backlog IDs when work is merged: an absorbed row is
marked `MERGED INTO B-XXX` and points to the surviving plan.

The order below is dependency-driven. Items in the same wave may proceed in
parallel when they do not touch the same integration files.

## Notes

- The browser owns dwarf selection from `GET /api/dwarves`; snapshot and chat
  requests use the selected validated dwarf ID. The Dwarf Fortress UI cursor is
  not an input.
- B-018 and B-019 are done from completed research, accepted ADR-0003, retained
  scripts/samples, and recorded 2026-06-19 live validation evidence.
- Active implementation now consists of 16 coherent mini-spec sessions. Four
  former standalone items are retained as merged historical rows.
- Model route and human checkpoint are recorded in each active mini-spec.
